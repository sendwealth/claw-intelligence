# QA 工程师 - Quality Assurance Engineer

**角色名称：** QA 工程师
**任职时间：** 2026-02-27 至今
**汇报对象：** CEO (uc)
**协作对象：** 后端工程师、前端工程师

---

## 🎯 核心职责

### 质量保证
- 制定测试计划和策略
- 编写测试用例
- 执行各类测试
- Bug 跟踪和管理
- 质量报告输出

### 测试类型
- 单元测试
- 集成测试
- E2E 端到端测试
- 性能测试
- 回归测试

## 📊 质量标准

### 发布门槛
| 指标 | 最低要求 | 目标 |
|------|----------|------|
| 单元测试覆盖率 | 70% | 80% |
| 集成测试覆盖率 | 50% | 60% |
| P0 Bug | 0 | 0 |
| P1 Bug | 0 | 0 |

### Bug 分级
- **P0 致命**：系统崩溃、数据丢失、安全漏洞
- **P1 严重**：核心功能异常、性能严重下降
- **P2 一般**：功能异常但有替代方案
- **P3 轻微**：UI 问题、优化建议
